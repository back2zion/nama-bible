# Phase 4: 데이터 품질 개선 (2026-03-22)

## 문제 발견

v3 학습 후 번역 품질을 분석하면서, 병렬 데이터에 심각한 노이즈가 있음을 발견.

### 분석 결과

| 문제 유형 | 건수 | 설명 |
|-----------|------|------|
| 멀티-절 합침 | 878 | 여러 절이 하나로 합쳐져 정렬 오류 발생 |
| 길이 비율 이상 (>5.0) | 542 | 나마어가 영어 대비 비정상적으로 김 |
| 길이 비율 이상 (<0.3) | 10 | 나마어가 영어 대비 비정상적으로 짧음 |
| **총 제거** | **1,430** | 전체 3,167 정렬쌍의 45% |

길이 비율 평균이 3.14에서 1.55로 정상화됨.

## 클리닝 기준

```python
# 너무 짧은 텍스트 제거
if len(nama) < 5 or len(eng) < 5: remove

# 길이 비율 이상치 제거
ratio = len(nama) / len(eng)
if ratio < 0.3 or ratio > 5.0: remove

# 멀티-절 패턴 감지
if re.search(r"\b\d{1,3}\s+[A-Z]", nama) and len(nama) > 300: remove
```

## 백트랜슬레이션 증강

- 원본 BT 데이터: 2,622쌍
- 멀티-절 필터링 후: 1,403쌍
- BT의 영어(역번역)를 영어 side로 사용 → 학습 데이터 증강
