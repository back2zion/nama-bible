# Phase 5: Human-in-the-Loop 번역 (2026-03-22)

## 개요

모델이 생성한 초안을 나마어 화자에게 보내고, 피드백(교정본)을 받아 학습 데이터에 추가하는 순환 구조를 구축했다.

```
모델 초안 생성 → 화자에게 전달 → 교정본 수신 → 학습 데이터 추가 → 재학습 → 개선된 초안
```

## 타임라인

1. **룻기 초안 생성**: TranslateGemma 27B로 85절 번역 (v3.1 모델)
2. **초안 전달**: 박정석 님을 통해 번역팀에 공유
3. **화자 번역 수신**: `룻기.rtf` — 나마어 화자가 직접 번역한 룻기
4. **RTF → USFM 변환**: 기존 파이프라인으로 처리 → `data/nmx/08-RUTnmx.usfm`
5. **비교 평가**: 모델 출력 vs 화자 번역

## 비교 결과

| Metric | 모델 vs 화자 |
|--------|-------------|
| BLEU   | 6.11        |
| chrF   | 37.06       |

### 샘플 비교 (룻기 1:11)

| | 텍스트 |
|---|---|
| **영어** | Naomi said, "Go back, my daughters..." |
| **화자** | Naomi nuwangoi, "Fém so néngangot, tane mérésof..." |
| **모델** | Naomi norayai yéfefaf, "Ngangotati, tane mérésmérés..." |

관찰:
- 핵심 문법 구조는 유사 (주어-동사-목적어)
- 어휘 선택 차이: `nuwangoi` vs `norayai` (둘 다 "말하다"의 변형)
- 모델이 나마어 형태소를 올바르게 사용하고 있음

## 데이터 추가

화자 번역 룻기 85쌍을 학습 데이터에 추가:
- `nama_eng_augmented.json`: 3,140 → **3,225쌍**
- 이것은 **최초의 구약 나마어 병렬 데이터**

## 재학습 (v3.2)

화자 피드백이 반영된 데이터로 모델을 재학습 중.
기대 효과:
- 구약 번역 시 화자의 표현 패턴을 더 잘 반영
- 다만 85쌍만으로 극적인 향상은 어려움 — 더 많은 구약 책의 화자 번역이 필요

## 다음 단계

1. 재학습 완료 후 룻기 재번역 → 화자 번역과 비교하여 개선 폭 측정
2. 다른 구약 책 (창세기, 출애굽기 등) 초안 생성 → 화자 피드백 루프 반복
3. 화자 교정본이 쌓일수록 모델 품질이 점진적으로 향상

## 핵심 인사이트

- 모델 초안은 **그 자체로 최종 결과물이 아니라**, 화자와의 피드백 루프를 촉발하는 도구
- 첫 번째 초안(chrF 37)이 완벽하지 않았지만, 이를 공유함으로써 화자 번역본을 확보
- 이 사이클이 반복될수록 모델 품질과 데이터 양이 동시에 증가하는 선순환 구조
