# Phase 1: RTF → USFM 변환 (2026-03-03)

## 배경

박정석 선교사님이 Paratext에서 내보낸 RTF 파일 4개를 제공해주셨다.
Paratext는 성경 번역 소프트웨어로, RTF 내보내기 시 USFM 마커를 RTF 스타일로 인코딩한다.
우리 파이프라인은 USFM 형식을 사용하므로 변환이 필요했다.

## RTF 파일 목록

| 파일 | 포함 책 | 최종 점검 |
|---|---|---|
| `마태복음부터 사도행전까지.rtf` | MAT, MRK, LUK, JHN, ACT (5권) | [x] |
| `로마서에서 고린도후서까지.rtf` | ROM, 1CO, 2CO (3권) | [ ] |
| `갈라디아서에서 골로새서까지.rtf` | GAL, EPH, PHP, COL (4권) | [x] |
| `데살로니가에서 요한계시록.rtf` | 1TH~REV (15권) | [ ] |

## Paratext RTF 구조

Paratext RTF는 일반 RTF가 아니라, USFM 구조가 그대로 스타일에 매핑되어 있다:

- **스타일시트**: `{\stylesheet ...}` 안에 `\s<번호>` → USFM 마커 매핑
  - `s0` → `\id` (책 식별)
  - `s52` → `\c` (장)
  - `s60` → `\p` (본문 단락)
  - `s105` → `\s` (섹션 제목)
  - `s93` → `\mt1` (메인 타이틀)
  - 총 128개 스타일 정의
- **본문**: `\pard\plain \s<번호>\...` 로 단락 구분
- **절 번호**: `{\cs57\...\fs22 <번호>\~}` 인라인 스타일
- **유니코드**: `\u233?` 형태 (é = \u233?)

## 변환 스크립트

```bash
uv run python scripts/rtf_to_usfm.py data/source_rtf/*.rtf -o data/nmx
```

변환 과정:
1. RTF 스타일시트 파싱 → 스타일번호:USFM마커 매핑 구축
2. 본문을 `\par\pard\plain` 기준으로 단락 분할
3. 각 단락의 스타일 번호로 USFM 마커 결정
4. `\cs57` 인라인 스타일 → `\v` (절 번호) 변환
5. `\u<코드>?` → 유니코드 문자 디코딩
6. 책 코드(`\id`) 기준으로 파일 분리

주의사항:
- 유니코드 디코딩을 RTF 명령어 제거보다 **먼저** 해야 함 (`\u`가 명령어로 인식되어 삭제됨)
- `\v` 마커는 placeholder(`\x00v`)로 보호 후 RTF 명령어 제거, 이후 복원

## 출력

- 위치: `data/nmx/`
- 파일명 패턴: `<번호>-<코드>nmx.usfm` (예: `41-MATnmx.usfm`)
- 신약 27권 전체 생성

## 기존 데이터와의 관계

- `data/nmx_ebible/`: 기존 USFM (LUK, ACT 2권만 있음, eBible.org 출처)
- `data/nmx/`: RTF에서 변환 (신약 27권, 박정석 선교사님 제공)
- 두 소스의 LUK/ACT는 내용이 동일하며, 서식 차이만 있음
